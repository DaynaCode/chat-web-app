# WebSocket API Documentation

## Endpoints

| Type  | URL                         |
|-------|-----------------------------|
| Chat  | `ws://<host>:8001/ws/chat/` |
| Status| `ws://<host>:8001/ws/status/` |

## Authentication

Add the JWT **access token** as a query parameter:

```
ws://api.photoshade.ir:8001/ws/chat/?token=eyJhbGciOi...
```

The token is validated on every connection.  
Refresh tokens are **not** accepted.

## 1. Chat WebSocket (`/ws/chat/`)

### Connection

* Automatically joins a group for each conversation the user is a member of (`conversation_{id}`).  
* Also joins a personal group (`user_{user_id}`) for private events like mentions and read receipts.  
* No manual room join is needed.

### Sending a Message

Send a JSON message with `type: "send_message"`:

```json
{
  "type": "send_message",
  "conversation_id": 1,
  "text": "Hello!",
  "image_id": null,
  "replied_to_id": null,
  "client_message_id": "unique-uuid-123"
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `type` | yes | Must be `"send_message"` |
| `conversation_id` | yes | ID of the conversation |
| `text` | no | Message text (can be empty if image is sent) |
| `image_id` | no | ID returned by `POST /api/upload/` |
| `replied_to_id` | no | ID of the message being replied to |
| `client_message_id` | no | A unique ID generated by the client to help deduplication and UI tracking |

### Receiving a New Message

All members of the conversation receive:

```json
{
  "type": "new_message",
  "message": {
    "id": 100,
    "conversation_id": 1,
    "sender": {
      "id": 1,
      "username": "admin"
    },
    "text": "Hello!",
    "image": null,
    "created_at": "2026-05-30T10:00:00Z",
    "client_message_id": "unique-uuid-123"
  }
}
```

### Mentions

When a message contains `@username`, the mentioned user receives a private event on their personal group:

```json
{
  "type": "mention",
  "message": {
    "id": 101,
    "conversation_id": 1,
    "sender": "admin",
    "text": "@ali check this",
    "created_at": "2026-05-30T10:01:00Z"
  }
}
```

### Error Messages

If message sending fails (e.g., not a member, validation error):

```json
{
  "type": "error",
  "detail": "You are not a member of this conversation."
}
```

## 2. Status WebSocket (`/ws/status/`)

### Connection

Automatically subscribes to `status_updates` group.  
When a user connects or disconnects, all clients receive:

```json
{
  "type": "user_status",
  "user_id": 2,
  "status": "online"
}
```

Possible statuses: `"online"`, `"offline"`.  
No client action needed; connect once and keep the socket open.

## 3. Read Receipts (REST + WebSocket)

* To mark messages as read, use the REST API:  
  `POST /api/read-receipts/` with `{"message_ids": [1,2,3]}`.
* The **sender** of those messages receives a WebSocket event on their personal group (`user_{sender_id}`):

```json
{
  "type": "read_receipt",
  "message_ids": [1,2,3],
  "reader_id": 2
}
```

## 4. Important Notes

* **Reconnection:** Implement exponential backoff. The token may expire after 15 minutes; you must refresh it via REST and reconnect with the new token.
* **Deduplication:** Use `client_message_id` to ignore duplicate delivery.
* **Conversation groups:** You do **not** need to send a join event. After authentication, the server subscribes you to all your existing conversations automatically.
* **New conversations:** If you create a new conversation via REST, you must reconnect the WebSocket to join its group (or implement a `join` event – not currently available).

## 5. Quick Example (JavaScript)

```javascript
const token = "YOUR_ACCESS_TOKEN";
const ws = new WebSocket(`ws://api.photoshade.ir:8001/ws/chat/?token=${token}`);

ws.onopen = () => {
  // Send a message
  ws.send(JSON.stringify({
    type: "send_message",
    conversation_id: 1,
    text: "Hello!",
    client_message_id: crypto.randomUUID()
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  switch (data.type) {
    case "new_message":
      console.log("New message:", data.message);
      break;
    case "mention":
      console.log("You were mentioned:", data.message);
      break;
    case "error":
      console.error("WebSocket error:", data.detail);
      break;
  }
};


| Event (Client → Server) | Payload | توضیح |
|--------------------------|---------|-------|
| `edit_message` | `{"type": "edit_message", "message_id": 1, "text": "متن جدید"}` | ویرایش متن پیام (فقط نویسنده) |
| `delete_message` | `{"type": "delete_message", "message_id": 1}` | حذف نرم پیام (نویسنده یا ادمین) |

| Event (Server → Client) | Payload | توضیح |
|--------------------------|---------|-------|
| `message_edited` | `{"type": "message_edited", "message": {"id": 1, "conversation_id": 1, "text": "متن جدید", "edited_at": "..."}}` | پیام ویرایش شد |
| `message_deleted` | `{"type": "message_deleted", "message_id": 1, "conversation_id": 1}` | پیام حذف شد |

---

**نکته:** پس از اعمال این تغییرات، هم از طریق REST (Viewها) و هم از طریق WebSocket (Consumers) رویدادهای ویرایش و حذف به‌صورت بلادرنگ به همهٔ اعضای مکالمه ارسال می‌شوند.
